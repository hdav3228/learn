Production choiceCurrent setupReason
Cohort separation One array for diabetes and one for non-diabetes. This follows the meeting plan and prevents output-path collisions.
Coverage Each cohort array covers all 4,018 genes exactly once. The final comparison needs one posterior sample per gene per cohort.
Concurrency At most 10 array tasks per cohort run at once. This keeps service-unit use and filesystem pressure under control.

10-Gene PPP Test

Why I tested it before the full run finished

I wanted to make sure the output format, burn-in handling, confidence interval calculation, comparisons, and plots all worked before thousands of traces arrived. I ran the pipeline on 10 early genes that had finished in both cohorts.

  1. Summary script. Reads the last omega0 column, calculates mean, median, and the 2.5th to 97.5th percentile interval, and flags unexpected trace lengths.
  2. Comparison script. Compares strict omega-above-1 hits against the Anthony thesis genes, top diabetes NS/S genes, and top DB-vs-NDB difference genes.
  3. Plot script. Makes one omega trace plot per cohort/gene and one heatmap per cohort.
Test resultWhat I foundInterpretation
Trace structure All 20 files had 1,600 rows, beginning at iteration 2,005 and ending at 10,000. The XML burn-in and thinning behaved as expected.
Positive-selection hits No gene/cohort trace had its full 95% omega interval above 1. This is a reasonable small-test result and does not establish the all-gene result.
Means above 1 Some posterior means were above 1, but their intervals still crossed 1. A mean above 1 by itself is not enough evidence for a strict hit.
Spreadsheet overlap The strict-hit comparison table had no rows because there were no strict hits. This is an expected empty result, not a comparison-script failure.
Plots The pipeline generated 20 trace plots and two cohort heatmaps. The plotting path is ready for the production outputs.
Main conclusion from the test
The small run did what it was supposed to do: it verified the engineering and analysis path. It was not expected to produce an important biological hit.

Questions To Ask

Decisions worth confirming with Dr. Ioerger

The production and first-pass summary paths are working. These questions determine what the final statistical comparison should look like after all genes finish.

  1. Ad-hoc ranking rule. Is the current pseudocount score acceptable, and should I rank observed per-isolate mutations, unique codon changes, or show both?
  2. DB-specific enrichment. For the first email question, should I prioritize a high DB NS/S score alone, or require that it is also clearly higher than NDB?
  3. Interval convention. I currently use the conventional 2.5th and 97.5th percentiles for a 95% interval. In the transcript, 5th to 95th was also said aloud, which would be a 90% interval. Which rule should I use in the final report?
  4. Hit definition. Should the first positive-selection list require the entire omega interval to be above 1, or should I also rank genes by posterior mean/probability above 1?
  5. Cohort comparison. After the separate runs, should I build a sampled diabetes-minus-non-diabetes omega difference distribution for each gene?
  6. Multiple testing. What ranking or false-discovery rule should I use before calling any all-gene result significant?
  7. Trace diagnostics. Are trace plots and row/final-iteration checks enough for the first pass, or should I add an ESS/autocorrelation threshold and rerun weak traces?
  8. Candidate follow-up. Should I prioritize genes such as Rv3104c and Rv3372 when the all-gene omega results arrive because they overlap Anthony's thesis list?
  9. Next model. After the constant-model comparison is complete, should I move to sliding-window analysis only for selected candidate genes?
