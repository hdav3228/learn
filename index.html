The role fit

You have a Stanford CS/statistics background. Treat this project as a large-scale Bayesian workflow engineering problem with a light biology wrapper: aligned DNA files go in, GenomegaMap posterior samples come out, and the analysis asks whether diabetes vs non-diabetes cohorts show different selection signals.

Plain English You are comparing two piles of TB genomes. For each gene, GenomegaMap estimates a distribution for ω (dN/dS), which is a selection score. Then you compare the diabetes distribution to the non-diabetes distribution and ask which genes look meaningfully different.

Use the constant GenomegaMap model first. It gives one gene-level ω posterior, which is the simplest scalable comparison across thousands of genes. Use the constant Bayesian XML template unless Dr. Ioerger approves different settings.
Run diabetes and non-diabetes separately. The first analysis is two posterior distributions per gene, one per cohort. Keep cohort labels explicit in file paths, result tables, and plots.
Filter bad isolates. Low-coverage or mixed-lineage samples can corrupt the comparison. Use only samplestatus=good: 178 diabetes and 175 non-diabetes samples.

Use Slurm for real compute, not the login node. Login nodes are for editing, transfers, and small preprocessing; compute nodes are scheduled through sbatch. Run XML/FASTA generation on the login node only because it is lightweight; run GenomegaMap through Slurm.

Every XML needs unique paths. Parallel jobs can accidentally read the same input or overwrite the same output. Verify one generated XML has the right filtered FASTA path and result path.

CS And Statistics Map

Use your existing strengths

Most of the biology words can be translated into familiar CS/statistics objects. Use this map when the vocabulary starts feeling like noise.

FASTA alignment

CS translation: a keyed table of strings where each row is an isolate and each column is an aligned genome position.

Why it matters: GenomegaMap compares columns/codons across samples, so alignment quality is data quality.

Gene-by-gene scan

CS translation: a map operation over ~4,018 independent input files per cohort.

Why it matters: Slurm arrays are the natural execution model, but each task needs isolated input/output paths.

Bayesian model

Stats translation: posterior is proportional to likelihood times prior.

Why it matters: you interpret a distribution over ω, not only a mean.

MCMC output

Stats translation: a dependent sample from a hard-to-compute posterior.

Why it matters: repeated rows, burn-in, and autocorrelation are diagnostics, not weird file noise.

Cohort comparison

Stats translation: compare two empirical posterior samples, e.g. Δω = ωD - ωND.

Why it matters: overlap and uncertainty matter more than one ranked point estimate.

Ad-hoc mutation tally

CS translation: parse codons, translate them, classify each observed change as synonymous or non-synonymous, then aggregate by gene and cohort.

Why it matters: this gives an independent ranking to compare against GenomegaMap before treating any hit as real.

HPRC Faster

Systems translation: a remote batch scheduler with a compute budget.

Why it matters: correctness, resource limits, logging, and restartability are part of the research method.

Biology minimum (30 seconds)Bio basics
Isolate
One patient's TB sample; each FASTA row is one isolate's copy of a gene.
Codon / dN/dS
DNA is read in 3-letter words; dN/dS compares protein-changing vs silent changes.
ω
Selection ratio. <1 purifying, ≈1 neutral, >1 positive.
Confounding
Lineage or resistance mix can fake a diabetes signal — treat as a stats design problem.

How to compare one gene after both cohorts finish

1
Read both trace files. One file is diabetes, one file is non-diabetes, both for the same gene.
2
Discard burn-in. Drop the early chain samples according to the setting Dr. Ioerger approves.
3
Summarize omega0. Compute mean, median, 95% credible interval, and probability that ω is above 1.
4
Sample a difference. Pair posterior samples to estimate P(omega_diabetes > omega_nondiabetes) and a credible interval for the difference.
