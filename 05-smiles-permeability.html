SMILES permeability: can a compound cross the mycobacterial cell wall?

This page turns the rough meeting notes into a concrete modeling workflow: DataWarrior molecular properties, Open Babel FP2 fingerprints, combined feature vectors, and direct SMILES-string models. The goal is not one magic classifier; it is a fair comparison with ROC curves, confusion matrices, and cross-validated accuracy around the published baseline.

In plain English A compound can bind a great target and still fail if it never gets into the TB cell. This task is the entry filter: given only the molecule, predict permeable vs not permeable before wasting docking or wet-lab effort on molecules that the cell wall is likely to reject.

1. The goal

Why this matters

Mycobacterium tuberculosis has an unusually difficult cell wall. The Radchenko et al. paper frames permeability as a binary classification problem over drug-like compounds, and reports cross-validated balanced accuracy of 0.768 and ROC AUC of 0.911 for its best fragment-descriptor neural net.

The local implementation is deliberately broader than one model. It asks five related questions:

QuestionModeling answerWhy it is useful
Are simple physicochemical properties enough? Naive Bayes, Random Forest, SVM, 2-layer MLP on DataWarrior descriptors. Fast, interpretable sanity check.
Do substructure fingerprints help? Run fp2feats.py, then train on 1024 FP2 bits. Fingerprints capture local fragments directly.
Do properties and fingerprints complement each other? Concatenate property columns with FP2 bits and cross-evaluate. This is often the strongest classical baseline.
Can the model learn from SMILES strings themselves? Character n-gram baseline, TensorFlow 1-D CNN, and transformer-ready script. Tests the motif-learning idea from SMILES CNN papers.
Can stronger chemistry features beat the 0.75-ish ceiling? RDKit descriptors, Morgan/ECFP4 fingerprints, MACCS keys, and an ensemble over old + RDKit views. This is the current best route: 0.787 balanced accuracy after the meta-ensemble combiner.
1. Start with molecules Each row is a compound with a SMILES string, ID, and permeable/not-permeable label.
2. Make feature views Properties, FP2, RDKit descriptors, Morgan fingerprints, and direct SMILES encodings.
3. Cross-evaluate Every model predicts held-out folds, so the confusion matrix is not training accuracy.
4. Pick the honest winner Best local route: RDKit component scores combined by a small meta-logistic model.

3. Feature pipelines

Plain English: the SMILES confusion
A SMILES string is the molecule written like a word. One-hot SMILES encoding turns each character position into spreadsheet columns. Fingerprints and descriptors are different: they are chemistry checklists, such as fragments, rings, atom paths, weight, polarity, and solubility. In these runs, the chemistry checklists beat the raw molecule spelling.

One molecule, three ways to show it to a model

Raw spelling

CC(=O)Nc1ccc... is compact, but the model must learn chemistry from characters.

One-hot table

Character position becomes columns like S017_04. Clean pipeline, weaker signal.

Chemistry checklist

Fingerprints/descriptors directly encode fragments, rings, paths, polarity, and weight.

SMILES-only: weaker CNN: learned signal RDKit/fingerprints: best

DataWarrior properties are fixed-length molecular descriptors: molecular weight, cLogP, cLogS, polar surface area, ring counts, rotatable bonds, donor/acceptor counts, and related chemistry summaries. They are compact enough for Gaussian naive Bayes, Random Forests, SVMs, and small MLPs. 

props = pd.read_csv("radchenko-cln.props.txt", sep="\t")
X_props = props.drop(columns=["id", "perm"])
y = props["perm"].astype(int)

FP2 is Open Babel's path-based fingerprint. The README's command converts SMILES into fingerprints, then fp2feats.py expands the hex output into 1024 binary columns. This directly represents fragment presence/absence. 

obabel -ismi radchenko-cln.txt -ofpt -Oradchenko-cln.fpt -xfFP2 -xh
python3 fp2feats.py radchenko-cln.fpt > radchenko-cln.fpt.feats.txt

The combined matrix concatenates molecular properties with FP2 bits. The properties give global chemistry; the fingerprints give fragment-level chemistry. In the local runs this is the representation that should be watched first. 

X_combined = props.merge(fp2, on="id")
feature_cols = [c for c in X_combined.columns if c not in {"id", "perm"}]

SMILES are variable-length strings, so the implemented scikit-learn baseline uses character n-grams as a motif detector. The TensorFlow script tokenizes each character and trains one-hot or embedding-based 1-D CNNs, with a small transformer implementation available for follow-up experiments. The standalone encoder can also export a fixed 38 x 200 one-hot SMILES matrix keyed by compound ID; train_encoded_smiles_6models.py uses that clean encoded table for a six-model scikit-learn check. 

TfidfVectorizer(
    analyzer="char",
    ngram_range=(2, 5),
    lowercase=False,
    min_df=2
)

Bottom line from the local runs
Adding RDKit chemistry features finally moved the needle. The best single non-RDKit model is the combined molecular-properties + FP2 Random Forest (0.769 balanced accuracy, 0.847 AUC). The previous ensemble reached 0.775 / 0.856. The RDKit-enhanced mean ensemble reaches 0.782 / 0.865 at the default threshold, or 0.785 balanced accuracy with the out-of-fold tuned threshold. A final meta-logistic combiner over the RDKit component scores is the current best local number: 0.787 balanced accuracy and 0.866 AUC. The direct SMILES 1-D CNN was tried and did not beat the fingerprint/property models. The new Ioerger-style encoded-SMILES six-model run is useful as a clean pipeline check, but its best model reached only 0.666 balanced accuracy and 0.734 AUC.
Do not oversell the last decimal
The jump from 0.785 to 0.787 is a useful local improvement, but a paired bootstrap check gave a 95% difference interval that crosses zero. Report it as the best current local score, not as a statistically secure breakthrough.

6. Direct SMILES models

Why this section matters

The SMILES CNN paper treats a SMILES string like a 1-D grid. A convolution filter can slide across characters and learn local motifs, analogous to motif detection in DNA sequence models.

The main Python 3.13 environment did not have TensorFlow, so a local Python 3.11 virtual environment was created at .venv-smiles/. The Hirohara-style one-hot 1-D CNN was tried with 5-fold cross-validation. It learned signal, but it did not beat fingerprints: 0.719 balanced accuracy and 0.793 ROC AUC.

The newer local check follows Ioerger's cleaner data flow: first export SMILES-only one-hot features from radchenko-cln.smi, then join the permeability labels by compound ID in the model script. Six ordinary scikit-learn models were run on that encoded table. ExtraTrees was best, at 0.666 balanced accuracy and 0.734 ROC AUC, which shows the pipeline works but the flattened one-hot representation loses chemistry that FP2/RDKit fingerprints preserve.
